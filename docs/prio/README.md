# Prioritization Docs

This folder holds planning, roadmap, and state-assessment material.

Files here are useful for prioritization and review, but they are not automatically the active execution source of truth.

Start with:
- `CURRENT_STATE_INVENTORY.md` for what is actually real now
- `ROADMAP.md` for the single canonical roadmap
- `SPRINT_01_SUMMARY_SURFACES_PLAN.md` if you are executing the first Overview + Wallboard sprint
- `CONSOLIDATED_BACKLOG.md` only if you want the intermediate consolidation history and backlog framing

Superseded planning inputs have been moved to `ARCHIVE/docs/prio/`.

Before acting on a file in `docs/prio/`:

1. compare it to the codebase
2. confirm what is still current
3. promote or rewrite only the valid parts
